## API Key Authentication

All API requests require an API key passed in the `X-API-Key` header.

```bash
curl https://api.wabridge.co/v1/tenants \
  -H "X-API-Key: wab_your_key"
```

Your API key is available in the [WaBridge Dashboard](https://wabridge.co). Keys are prefixed with `wab_`.

<Warning>
  Keep your API key secret. Never expose it in client-side code or commit it to version control.
</Warning>

## Inbox Token (JWT)

For embedding the inbox component in your frontend, use a short-lived JWT token instead of the API key.

Generate a token server-side:

```bash
curl -X POST https://api.wabridge.co/v1/tenants/TENANT_ID/inbox-token \
  -H "X-API-Key: wab_your_key"
```

Response:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "expires_at": "2026-03-28T10:00:00Z"
}
```

The token:
- Expires after **24 hours**
- Is scoped to a single tenant
- Should be generated server-side and passed to your frontend

## Error Responses

Invalid or missing API keys return a `401` error:

```json
{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or has been revoked"
  }
}
```

## Tenant Access Control

Every request that operates on tenant data validates that the tenant belongs to your app. Attempting to access another app's tenant returns `403`:

```json
{
  "error": {
    "code": "TENANT_NOT_FOUND",
    "message": "Tenant not found or does not belong to this app"
  }
}
```
